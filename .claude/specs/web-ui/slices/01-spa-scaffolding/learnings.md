# Learnings: SPA Scaffolding (Slice 01)

## Implementation Notes

### `package-lock.json` must be committed

The `build/web/makefile` uses `npm ci`, which requires a `package-lock.json` to exist.
The plan didn't mention this, but the lockfile must be generated (`npm install`) and
committed alongside `package.json`. Without it, `make web.build` fails immediately with:

> The `npm ci` command can only install with an existing package-lock.json

**Action taken:** Ran `npm install` to generate `package-lock.json`; it must be included
in the commit.

### `globals` package needed explicitly

The standard Vite React-TS scaffold's `eslint.config.js` imports `globals` for browser
globals (`globals.browser`). This package is not a transitive dependency of any listed
dev dep, so it must be declared explicitly in `devDependencies`. The plan omitted it.

**Action taken:** Added `"globals": "^15.11.0"` to `devDependencies`.

### Shell working directory persists in Bash tool across calls

The Bash tool shell state persists within a session (working directory carries over).
After `npm install` ran in `src/Worms.Hub.Web`, subsequent bare `npx` calls also ran
there. This is expected but worth noting when verifying commands against expected paths.

### `make web.lint` runs silently on success

The `@echo ""` at the end of `web.lint` means a successful run produces no visible
output. This is consistent with the other make targets in the project.

### Build output path works as-is

Setting `build.outDir: '../../.artifacts/web/'` in `vite.config.ts` correctly places
the bundle at `.artifacts/web/` relative to the repo root when building from
`src/Worms.Hub.Web/`.
