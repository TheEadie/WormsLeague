# Plan: Mockup Alignment

## Context

This slice restyles the landing page, header, and footer of the SPA at `src/Worms.Hub.Web/` to match the Claude Design mockups in `.claude/specs/web-ui/design/`. It establishes the shared chrome (brand block, sticky paper-background AppBar, bordered footer, JetBrains Mono accent typography) that subsequent authenticated pages will sit inside. It builds on slice 05 (the landing page with `Layout`/`Header`/`Footer`/`LandingPage` + `worm.png` already in `public/`) and slice 06 (the MUI CSS-variables theme with `light`/`dark` colour schemes plus `ColourSchemePicker`). The page remains read-only — the existing `<Button>Sign in</Button>` keeps no `onClick` wiring; that is the next slice (Browser sign-in).

This slice deliberately does **not** introduce any of the authenticated-only chrome from the mockup (breadcrumbs, league menu, league-scoped nav items, total-matches chip, profile chip), nor the landing page's stats strip, status chip, version caption, "Replay uploader" row, Privacy/House rules links, body paragraph beneath the headline, or the WormBobber SVG.

## Implementation Details

### 1. JetBrains Mono webfont

Use the `@fontsource/jetbrains-mono` npm package (SIL OFL 1.1). Self-hosting via Fontsource avoids an external CDN dependency, keeps the asset graph inside the Vite bundle, and matches Vite-native usage patterns.

Exact version: `^5.2.5` (current stable on the 5.x line).

Install:

```
cd src/Worms.Hub.Web
npm install @fontsource/jetbrains-mono@^5.2.5
```

Both `package.json` and `package-lock.json` must be committed.

In `src/main.tsx`, add these imports at the top (after the React imports, before the MUI imports — keeping ESLint's import order happy):

```ts
import '@fontsource/jetbrains-mono/400.css'
import '@fontsource/jetbrains-mono/500.css'
import '@fontsource/jetbrains-mono/700.css'
```

Only the three weights actually used by the mockup (400, 500, 700) are imported to keep the bundle slim.

Create `src/Worms.Hub.Web/THIRD_PARTY_NOTICES.md` with this content:

JetBrains Mono is licensed under the SIL Open Font License, Version 1.1.

This `THIRD_PARTY_NOTICES.md` provides the user-discoverable attribution.

### 3. Theme extraction (`src/theme.ts`)

Move the `createTheme` call out of `main.tsx` so the JetBrains Mono stack is reused without string duplication.

```ts
import { createTheme } from '@mui/material/styles'

const monoFontFamily = '"JetBrains Mono", ui-monospace, SFMono-Regular, Menlo, Consolas, monospace'

const theme = createTheme({
    cssVariables: { colorSchemeSelector: 'data-mui-color-scheme' },
    colorSchemes: { light: true, dark: true },
    typography: {
        // MUI default Roboto/system stack is kept for body text.
        // Re-skin the `overline` variant as our accent caption so the
        // "LEAGUE ACCESS" divider and any future similar callouts inherit
        // it consistently.
        overline: {
            fontFamily: monoFontFamily,
            fontWeight: 500,
            letterSpacing: '0.16em',
        },
    },
})

export { theme, monoFontFamily }
export default theme
```

`monoFontFamily` is exported so the `Header` wordmark (which is a `Typography` with custom inline styling, not an `overline`) can use the same stack without string duplication.

### 8. `LandingPage.tsx` rewrite

Two-column grid, hero on the left, sign-in card on the right. On `xs` it collapses to a single column (hero on top, card below) — this matches the spec's "stack vertically (hero on top, sign-in card below)" criterion.

```tsx
// Image source: https://static.wikia.nocookie.net/oneyplays/images/5/52/Worm.png — Worms Armageddon asset (Team17)
import Box from '@mui/material/Box'
import Button from '@mui/material/Button'
import Divider from '@mui/material/Divider'
import Paper from '@mui/material/Paper'
import Stack from '@mui/material/Stack'
import Typography from '@mui/material/Typography'

function LandingPage() {
    return (
        <Box
            sx={{
                display: 'grid',
                gridTemplateColumns: { xs: '1fr', md: '1.15fr 0.85fr' },
                minHeight: { md: 'calc(100vh - 52px)' },
            }}
        >
            {/* Hero column */}
            <Box
                sx={{
                    p: { xs: 4, md: 8 },
                    display: 'flex',
                    flexDirection: 'column',
                    justifyContent: 'center',
                    gap: 4,
                    borderRight: { md: 1 },
                    borderColor: { md: 'divider' },
                }}
            >
                <Box
                    component="img"
                    src="/worm.png"
                    alt="Worm"
                    sx={{ maxWidth: 240, width: '100%', height: 'auto' }}
                />
                <Typography
                    variant="h2"
                    component="h1"
                    sx={{ fontWeight: 800, lineHeight: 1.0, letterSpacing: '-0.02em' }}
                >
                    Every shot. Every kill. Archived.
                </Typography>
            </Box>

            {/* Sign-in column */}
            <Box
                sx={{
                    p: { xs: 4, md: 8 },
                    display: 'grid',
                    placeItems: 'center',
                    bgcolor: 'background.default',
                }}
            >
                <Paper
                    elevation={4}
                    sx={{ width: '100%', maxWidth: 400, p: 4, borderRadius: 3 }}
                >
                    <Typography variant="h5" sx={{ fontWeight: 700, mb: 0.5 }}>
                        Sign in to continue
                    </Typography>
                    <Typography variant="body2" color="text.secondary" sx={{ mb: 3 }}>
                        League members only.
                    </Typography>

                    <Button variant="contained" fullWidth size="large" sx={{ py: 1.5 }}>
                        Sign in
                    </Button>

                    <Divider sx={{ my: 3 }}>
                        <Typography variant="overline" color="text.secondary">
                            League access
                        </Typography>
                    </Divider>

                    <Stack
                        direction="row"
                        sx={{ justifyContent: 'space-between', alignItems: 'center' }}
                    >
                        <Typography variant="body2" color="text.secondary">
                            Need an invite?
                        </Typography>
                        <Typography variant="body2" color="primary.light">
                            Contact Eadie on Slack
                        </Typography>
                    </Stack>
                </Paper>
            </Box>
        </Box>
    )
}

export default LandingPage
```

Notes:
- The `<Button>` deliberately has **no** `onClick` — spec requires it be non-functional in this slice.
- The image-source attribution comment at the top of the file is preserved from the existing implementation.
- The overline variant inherits the JetBrains Mono + uppercase + letter-spacing treatment from `theme.ts`, so the "League access" caption gets the mockup's accent typography automatically. (MUI's default `overline` already does `textTransform: 'uppercase'`.)
- `borderRight` is omitted on `xs` so the stacked layout looks clean.
- The sign-in column's `bgcolor: 'background.default'` differs from the AppBar's `background.paper`, giving the same subtle two-tone the mockup uses (this works correctly under both light and dark palettes since both tokens are palette-aware).

### 9. Ordering of changes

1. `npm install @fontsource/jetbrains-mono@^5.2.5` (regenerates lockfile).
2. Add `THIRD_PARTY_NOTICES.md`.
3. Add `src/theme.ts`.
4. Update `src/main.tsx` to import the new theme + fontsource CSS.
5. Add `src/components/Brand.tsx`.
6. Rewrite `src/components/Header.tsx`.
7. Rewrite `src/components/Footer.tsx`.
8. Rewrite `src/pages/LandingPage.tsx`.
9. Run `npx prettier --write src` so any reformatting lands before the lint pass (Prettier reformatted code in slice 05; same risk here).
10. Run `make web.build` then `make web.lint` from the repo root.

### 10. Caveats and risks

- **Prettier may reformat the JSX.** Slice 05's learnings flagged this. Run `npx prettier --write src` before committing or before the lint pass so the diff is what Prettier wants.
- **`make web.lint` runs `web.build` first in CI** because `npm ci` is gated on `package-lock.json`. Locally, run `web.build` once after `npm install` so `node_modules` is consistent with the new lockfile.
- **No CI changes needed.** The web build/lint jobs are unchanged; the new font is a runtime dependency installed via the existing `npm ci` step.
- **No Gateway changes.** Spec explicitly excludes API work.
- **No Docker/nginx changes needed.** The font files become part of the Vite bundle (Fontsource CSS references `url()` paths that Vite rewrites into hashed assets under `assets/`). The existing nginx config serves `assets/*` unchanged. No new whitelist entries are required in `build/web/Dockerfile.dockerignore`.
- **`worm.png` is reused as-is.** The slice does not introduce new image assets; the existing `public/worm.png` covers both the hero column and the header brand block.
- **The hero column's image source comment** must be preserved when the file is rewritten — it is the existing attribution for the worm image and is unrelated to the JetBrains Mono attribution.
