# Plan: Dark Mode

## Context

This slice adds OS-preference-aware dark/light theming to the web UI, with a three-way toggle (light / dark / system) in the Header. It builds directly on the public landing page slice (05), which established `ThemeProvider` + `CssBaseline` in `main.tsx`, the `Header` component (currently showing only the title), and the `Layout` / `LandingPage` components.

MUI v9 (already at `9.0.1`) ships first-class CSS-variables-based color scheme support. When a theme is created with `cssVariables: true` and `colorSchemes: { light: true, dark: true }`, the `ThemeProvider` automatically reads and writes the chosen mode to `localStorage` (default key `mui-mode`), responds to the `prefers-color-scheme` media query when mode is `system`, and exposes `useColorScheme()` for reading and setting the current mode. No external state management or manual `localStorage` reads are needed â€” MUI handles all of it.

The `@mui/icons-material` package (not yet installed) provides the `LightMode`, `DarkMode`, and `SettingsBrightness` (half-sun/half-moon, representing "system") icons required by the spec.


`@mui/icons-material` v9 is the correct version for `@mui/material` v9; the package follows MUI's unified major-version cadence.

### 2. Switching `main.tsx` to a CSS-variables theme

Replace the current `createTheme()` call and `ThemeProvider` usage:

```tsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import { createTheme, ThemeProvider } from '@mui/material/styles'
import CssBaseline from '@mui/material/CssBaseline'
import App from './App.tsx'

const theme = createTheme({
    cssVariables: true,
    colorSchemes: { light: true, dark: true },
})

createRoot(document.getElementById('root')!).render(
    <StrictMode>
        <ThemeProvider theme={theme} defaultMode="system">
            <CssBaseline />
            <App />
        </ThemeProvider>
    </StrictMode>,
)
```

Key points:
- `cssVariables: true` enables the CSS-variables engine that backs `useColorScheme`.
- `colorSchemes: { light: true, dark: true }` declares both palettes using MUI's built-in defaults â€” no custom brand colours.
- `defaultMode="system"` means the OS preference is applied when `localStorage` has no stored value, satisfying the "defaults to OS preference" acceptance criteria.
- `ThemeProvider` manages `localStorage` persistence under key `mui-mode` automatically; the spec accepts MUI's default storage key.

### 3. `ColourSchemePicker` component

Create `src/Worms.Hub.Web/src/components/ColourSchemePicker.tsx`:

```tsx
import { useColorScheme } from '@mui/material/styles'
import IconButton from '@mui/material/IconButton'
import Tooltip from '@mui/material/Tooltip'
import LightModeIcon from '@mui/icons-material/LightMode'
import DarkModeIcon from '@mui/icons-material/DarkMode'
import SettingsBrightnessIcon from '@mui/icons-material/SettingsBrightness'

function ColourSchemePicker() {
    const { mode, setMode } = useColorScheme()

    return (
        <>
            <Tooltip title="Light">
                <IconButton
                    onClick={() => setMode('light')}
                    color={mode === 'light' ? 'inherit' : 'default'}
                    aria-label="Light mode"
                    aria-pressed={mode === 'light'}
                >
                    <LightModeIcon />
                </IconButton>
            </Tooltip>
            <Tooltip title="System">
                <IconButton
                    onClick={() => setMode('system')}
                    color={mode === 'system' ? 'inherit' : 'default'}
                    aria-label="System mode"
                    aria-pressed={mode === 'system'}
                >
                    <SettingsBrightnessIcon />
                </IconButton>
            </Tooltip>
            <Tooltip title="Dark">
                <IconButton
                    onClick={() => setMode('dark')}
                    color={mode === 'dark' ? 'inherit' : 'default'}
                    aria-label="Dark mode"
                    aria-pressed={mode === 'dark'}
                >
                    <DarkModeIcon />
                </IconButton>
            </Tooltip>
        </>
    )
}

export default ColourSchemePicker
```

Design notes:
- `useColorScheme()` is only valid inside a tree wrapped by a `ThemeProvider` with `cssVariables: true` (which `main.tsx` now provides). Calling it outside that context throws; the component is only ever rendered inside `Layout` which is inside `App` which is inside the provider.
- `color={mode === 'x' ? 'inherit' : 'default'}` is the active-state visual indicator. `'inherit'` on an `IconButton` inside an `AppBar` uses the AppBar's contrasting text colour (fully opaque), whereas `'default'` renders at reduced opacity â€” this gives a clear selected/unselected distinction without introducing custom colours.
- `aria-pressed` follows ARIA button pattern for toggled states; improves accessibility.
- `Tooltip` labels ("Light", "System", "Dark") aid discoverability.
- The order light â†’ system â†’ dark is left-to-right, matching the natural light-to-dark spectrum. The spec does not specify order, so this is a reasonable default.
- `mode` can be `undefined` on the server / before hydration, but this is a pure-client SPA (no SSR), so `undefined` will not appear in practice. No additional null guard is needed.
